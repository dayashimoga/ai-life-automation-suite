# AI Life Automation Suite: Detailed Requirements

## 1. System Vision
The core directive of the AI Life Automation Suite is to implement three wholly independent, modular FastAPI applications capable of functioning locally and seamlessly without overlapping dependencies or environment pollution. 

## 2. Multi-Modal Memory Journal
### Functional Requirements
- **Blob Ingestion**: Upload native `.jpg` images and raw `.mp4` video frameworks payloaded via modern PWA boundaries.
- **REST Telemetry**: Dynamically strip MP4 keyframes via OpenCV and assert HTTPX pipelines to the remote Vision API server for automated background metadata tracking.
- **Sentiment & Transcription**: Formulate a Mock LLM payload to generate simulated Audio transcipts and memory block sentiment analyses without massive open-source models blocking RAM capabilities.
- **Database Architecture**: Non-ephemeral memory block lists mapping to frontend React-style Timeline states.

## 3. Doomscroll Breaker AI
### Functional Requirements
- **Threshold Blockers**: Allow users to query an active `AppUsage` telemetry payload, logging duration. If total bounds exceed constraints, inject highly-visible Focus Sessions.
- **Predictive AI LSTM Matrix (Phase 3)**: Deploy a robust Heuristic matrix assessing Time-of-Day Risk Indices.
- **Preemptive Sandboxing**: If the Doomscroll Predictor routes a >75% risk variable (e.g. tracking Instagram at 2:00 AM), forcefully route the user into a 15-Minute focus block locking out the DOM *prior* to behavioral spirals.

## 4. Visual Intelligence Matrix
### Functional Requirements
- **Live Stream Tracking**: WebRTC canvas manipulation ingesting native `getUserMedia` video and transmitting bytes globally.
- **WebSocket Streaming (Phase 3)**: Broadcast detection payloads and events over a Zero-Latency Pydantic Websocket endpoint to eliminate standard UI `setInterval` pooling latency.
- **Biometric Security Engine**: Run cropped YOLOv8 persons classes through FaceNet `DeepFace` arrays to verify Authorized vs Unknown biometric presence arrays natively.
- **Deep Historical Storage**: Inject every individual camera ping natively into a locally bound SQLite memory file, accessible via the web history API endpoints.

## 5. Security and Android PWAs
- Absolutely NO external environment bloat. `pip` packages strictly confined to `apps/*APP*/.venv_tmp` silos.
- 100% Android PWA compliance. Local `manifest.json` configurations dynamically injected across all three HTML deployments spanning offline cache telemetry.
